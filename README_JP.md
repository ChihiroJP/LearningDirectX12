## DirectX 12 ポートフォリオプロジェクト — Grid Gauntlet

**DirectX 12 ゲームプロジェクト** — カスタムレンダリングエンジンとモダンなグラフィックス機能を搭載し、その上に構築されたタクティカルパズルゲーム：**Grid Gauntlet**。

### プロジェクト紹介：Grid Gauntlet

**サイバー/ネオン美学**を持つ、危険なグリッド上の**タクティカル貨物プッシュパズルゲーム**。

- **目的**：貨物キューブをグリッドの左端から右端まで押して移動させる
- **操作**：WASDで貨物を押す（プレイヤーが貨物に向かって歩くと押す）、E + WASDで貨物を引く（壁に詰まった状況を解決）。Xboxコントローラー対応（左スティック + Aボタン）
- **貨物の落下**：貨物をグリッドの端から押し出すとステージから落下 — ステージ失敗
- **危険**：グリッド周囲のタワーがパターン化された攻撃を発射（行掃射、列攻撃、範囲爆発、追尾弾）、影響を受けるタイルに予告警告表示
- **パズル**：一部の通路は壁で塞がれており、タワーの攻撃を壁に誘導することでのみ破壊可能
- **ハザード**：炎（継続ダメージ）、雷（周期的バースト）、スパイクトラップ（ダメージ + スタン）、氷（スロー）、崩壊タイル（踏んだ後に崩壊）
- **進行**：25ステージ（開発予定）、全ステージ解放済み、可変グリッドサイズ（10x10〜50x50）、カウントアップタイマーとS/A/B/C評価
- **ビジュアル**：プロシージャルジオメトリ（立方体、円錐、円柱）+ エミッシブマテリアル + ブルーム — 外部3Dアセット不要

### エンジン機能
- ディファードレンダリング（Gバッファ：アルベド、法線、メタリック/ラフネス、エミッシブ）
- PBRマテリアル（メタリック-ラフネスワークフロー）+ ノーマルマッピング、視差オクルージョンマッピング
- カスケードシャドウマップ（4カスケード）
- イメージベースドライティング（IBL）HDRIスカイボックスから
- ポストプロセス：ブルーム、被写界深度、モーションブラー、TAA、FXAA、SSAO、トーンマッピング
- パーティクルシステム（CPUビルボード、GPUレンダリング）
- インスタンスメッシュレンダリング
- プロシージャルタイルシェーダー（アニメーションHLSLノイズ：溶岩、氷、雷、スパイク、崩壊）
- 複数ポイント/スポットライト（各32灯、ディファード）
- Dear ImGui デバッグUI
- 解像度対応（フルスクリーン、ボーダーレス、ウィンドウ 720p/1080p/1440p/4K）
- TPSカメラ（スムースフォロー、壁衝突オートズーム、ダメージ時の画面揺れ）
- Xboxコントローラー対応（XInput）

### 必要環境
- Windows 10/11
- Visual Studio 2022以上（MSVCツールチェーン）+ **Windows 10/11 SDK**
- CMake 3.24以上

### ビルドと実行
```powershell
cmake -S . -B build -G "Visual Studio 18 2026" -A x64
cmake --build build --config Debug
.\build\bin\Debug\DX12Tutorial12.exe
```

### プロジェクト構成
```
src/                    C++ソース（エンジン + ゲーム）
  gridgame/             Grid Gauntlet ゲームコード
  game/                 レガシーゲームコード（ビルドから切り離し済み）
shaders/                HLSLシェーダー（ランタイムコンパイル）
notes/                  学習ノート / アーキテクチャドキュメント
```

### 実行に関する注意事項
- **シェーダー**は `shaders/` 配下にあり、`D3DCompileFromFile` を使用してランタイムでロード/コンパイルされます。
- ビルド時に `shaders/` が `.exe` の隣にコピーされます。Visual Studioから実行する場合は、**作業ディレクトリ**を出力フォルダに設定するか、`build/bin/<config>/` から直接exeを実行してください。
